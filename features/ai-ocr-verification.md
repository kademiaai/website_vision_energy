# Feature: AI-Powered ID Verification (OCR)

## 1. Objective
Automate the extraction of sensitive identity information from Vietnamese Citizen ID Cards (CCCD) to facilitate reward fulfillment while minimizing manual data entry and human error.

## 2. Technical Workflow
1.  **Identity Handshake**: 
    - User accesses a unique link `/rewards/[token]`.
    - User must verify their **License Plate** and **Phone Number** against the record in the database before the OCR interface is revealed.
2.  **Image Processing**:
    - User captures/uploads a photo of their CCCD.
    - **Client-side Resizing**: Images are resized to < 1MB and converted to Base64 using `lib/imageUtils.ts` to ensure fast upload and reduced AI processing costs.
3.  **AI Extraction**:
    - Server Action: `app/services/ocrAction.ts`.
    - Engine: **Puter AI** (REST API).
    - Model: Primary `gpt-4o-mini`, Fallback `mistral`.
    - Output: Strictly formatted JSON containing `full_name` and `id_number`.
4.  **Data Persistence**:
    - Extracted data is auto-filled into a confirmation form.
    - The ID photo is uploaded to a private Supabase Storage bucket (`verification-docs`).
    - The `rewards` table is updated to `processing` status.

## 3. Security & Privacy
- **Private Storage**: ID photos are NOT public. They are stored in a protected bucket.
- **Access Control**: Only administrators can view these photos via short-lived (5-minute) signed URLs generated by the server.
- **Data Minimization**: The AI only extracts the name and ID number; other sensitive data on the card is ignored by the prompt.
- **Token Security**: Tokens are generated using `crypto.randomUUID()` making them virtually impossible to guess.

## 4. Implementation Details
- **Prompt Engineering**: The system uses a specific prompt to handle Vietnamese characters and enforce ALL CAPS for names.
- **Parsing Robustness**: Includes logic to clean AI responses (removing markdown code blocks) to ensure `JSON.parse()` never fails.
- **Fallback Logic**: If the primary vision model fails or returns a non-200 status, the system automatically retries with a secondary model.

## 5. Future Enhancements
- **Face Matching**: Verify the photo on the CCCD matches a "selfie" captured during the claim process.
- **Auto-Redaction**: Redact sensitive areas of the CCCD (like home address) before storage.
